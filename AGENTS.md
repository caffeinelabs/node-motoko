# AGENTS.md

`motoko` is a TypeScript library that compiles and runs Motoko smart contracts in Node.js or the browser by wrapping a bundled `moc` compiler.

## Commands

- Build: `npm run build` (cleans `lib/` then runs `tsc -p .`).
- Test: `npm test` (Jest via `ts-jest`).
- Format: `prettier --write <files>` (no dedicated script; config in `.prettierrc`).
- Bundle size: `npm run size-limit`.
- Regenerate bundled compiler/core/error codes: `npm run generate` (prints latest versions), then `npm run generate <moc_version> <core_version>`.

There is no lint script; formatting is the only style check.

## Layout

- `src/` — TypeScript source; the only directory compiled to `lib/`.
- `lib/` — build output of `tsc`; generated, do not edit (gitignored).
- `versions/latest/` — bundled `moc.min.js` / `moc_interpreter.min.js`, generated by `npm run generate`; do not hand-edit.
- `packages/latest/core.json` — bundled Motoko core library, generated; do not hand-edit.
- `contrib/generated/errorCodes.json` — generated by `npm run generate`; do not hand-edit.
- `contrib/` — editor/highlighting integrations (hand-written: `hljs.js`, `monaco.js`, `snippets.json`).
- `utils/generate.js` — the script behind `npm run generate`.
- `examples/` — runnable usage samples. `tests/` — Jest specs.

## Conventions & gotchas

- Prettier config: 4-space tabs, single quotes, semicolons, trailing commas (`all`). Applied to `lib`, `contrib`, `utils`.
- A Husky `pre-commit` hook runs `lint-staged`, which formats staged JS/TS in `lib`, `contrib`, `utils`.
- `.npmrc` sets `min-release-age=7`; dependency installs respect a 7-day minimum release age.
- Tests run on Node.js 22.x and 24.x in CI; publish and generate workflows use Node 24.
- CI runs `npm ci`, `npm run build`, `npm test` on pushes/PRs to `main`.
- Publishing is automatic: pushing a changed `version` in `package.json` to `main` builds, tests, and publishes to npm.
- The `generate` script reads a Motoko checkout for error codes; override its location with the `MOTOKO_REPO` environment variable (defaults to `../../motoko/`).
